# 🎯 エディターUI改修指示書 — レイアウト全面リニューアル

## 概要

エディター画面のレイアウトを全面的に改修します。
**目的：初回ユーザーの「何をすればいいか分からない」問題を解決し、初回生成率を上げる。**

現在の3ペイン構成（サイドバー + 設定パネル + プレビュー）を、
**左パネル（入力・設定・生成ボタン）+ 右パネル（生成結果の表示のみ）** のシンプルな2ペイン構成に変更します。

**ブランチ**: `feature/editor-redesign` で作業し、完了後にPRを出してください。

## 🔧 左パネルの詳細構成

左パネルはスクロール可能で、以下の順序で要素を配置：

### 1. モード切替タブ

```
┌──────────┬──────────┬──────────┬──────────┐
│ Text to  │ Image to │  Face    │          │
│  Image   │  Image   │  Swap    │ Inpaint  │
└──────────┴──────────┴──────────┴──────────┘
```

- 現在のサイドバーメニュー（Text to Image / Image to Image / Face Swap / Inpaint）を **左パネル上部のタブ** に移動
- アクティブタブはハイライト表示
- タブ切替で、下の入力エリアが動的に変わる

### 2. 入力エリア（モードによって変化）

#### Text to Image モード

```
┌─────────────────────────────────┐
│ 🪄 想像しているイメージを         │
│    書いてみてください             │
│                                 │
│                    ✏️ 最適化     │
└─────────────────────────────────┘
```

- テキストエリア（3〜5行分の高さ）
- 右下に「最適化」ボタン（既存機能）
- **プレースホルダーをローテーション表示**（後述）

#### Image to Image モード

```
┌────────────────┬────────────────┐
│  📂 ライブラリ  │  ⬆ アップロード │
└────────────────┴────────────────┘
┌─────────────────────────────────┐
│ 参照画像をベースに、想像している   │
│ シーンを書いてみてください        │
│                                 │
└─────────────────────────────────┘
```

- 画像アップロードエリア + テキストエリア
- 添付画像2のUIを参考に

#### Face Swap モード

```
┌─────────────────────────────────┐
│  ⬆ ベース画像をアップロード       │
└─────────────────────────────────┘
┌─────────────────────────────────┐
│  ⬆ 顔画像をアップロード          │
└─────────────────────────────────┘
```

#### Inpaint（Nude Mode）モード

```
┌─────────────────────────────────┐
│  ⬆ 画像をアップロード            │
└─────────────────────────────────┘
│  🖌️ マスク範囲を指定             │
```

### 3. 設定セクション（折りたたみ可能）

```
┌─ 設定 ──────────────────── ▼ ──┐
│                                │
│  モデル    [RealVisXL V5.0 ▼]  │
│  解像度    [512] [1024🔒] ...  │
│  スタイル  [Film][Dreamy]...   │
│                                │
└────────────────────────────────┘
```

- **デフォルトは折りたたみ状態**（初回ユーザー向け）
- クリックで展開
- 上級ユーザーは自分で開いて調整
- 展開/折りたたみ状態はlocalStorageで記憶

### 4. Character Tags セクション（折りたたみ可能）

```
┌─ Character Tags ────────── ▼ ──┐
│                                │
│  AGE:   [Early 20s][Late 20s]  │
│  PEOPLE: [1][2][Multiple]      │
│  ETHNICITY: [Asian][European]  │
│  ...                           │
│                                │
└────────────────────────────────┘
```

- これも **デフォルトは折りたたみ状態**
- 展開/折りたたみ状態はlocalStorageで記憶

### 5. 生成ボタン（常に表示・左パネル下部に固定）

```
┌─────────────────────────────────┐
│                                 │
│     [  ✨ 生成  ✦ 2 credits ]   │
│                                 │
│     推定生成時間: 10s 〜 41s      │
│                                 │
└─────────────────────────────────┘
```

- **左パネルの下部に `position: sticky` で固定**
- スクロールしても常に見える
- クレジット消費量を明示

### ワンクリック生成の動作

ボタンを押すと以下が自動的に実行される：

```tsx
const ONE_CLICK_PRESET = {
  prompt: "beautiful woman, beach, golden hour, bikini, wind in hair, photorealistic",
  model: "RealVisXL V5.0",
  aspectRatio: "9:16",
  resolution: "512",
  nude: true,
  style: "natural",
  tags: {
    age: "early_20s",
    ethnicity: "asian",
    people: "1",
  },
};
```

- プロンプト欄にプリセットが自動入力される
- 設定が自動適用される
- **即座に生成が開始される**
- 生成中は右パネルにローディングアニメーションを表示

### サンプルプロンプトの動作

💡のプロンプト例をクリックすると：

- プロンプト欄にそのテキストが自動入力される
- ユーザーは必要に応じて修正してから生成ボタンを押す
- **自動生成はしない**（ワンクリックとの差別化）

## 📝 プロンプト入力欄のプレースホルダーアニメーション

プレースホルダーテキストが3秒ごとに切り替わる：

```
例1: "ビーチで赤いビキニの美女"
  ↓ 3秒後
例2: "カフェで笑顔のワンピース女性"
  ↓ 3秒後
例3: "夜景をバックにドレスの美女"
  ↓ 3秒後
例4: "beautiful woman in red dress, city night"
  ↓ ループ
```

フェードイン/フェードアウトのアニメーションで切り替え。

## ⚠️ 注意事項

1. **既存機能を壊さない** — 各モードの生成ロジック、API呼び出し、クレジット計算はそのまま維持。変更するのはUIレイアウトのみ。

2. **設定の初期値** — 折りたたみ状態の初期値は `collapsed`。`localStorage` に `editor.settings.expanded` と `editor.tags.expanded` で記憶。

3. **ワンクリック生成はクレジットを消費する** — 通常の生成と同じ扱い。無料クレジットから引かれる。

4. **ウェルカムカードは1回目の生成後に非表示** — 生成履歴が1件以上ある場合は通常のギャラリーを表示。

5. **i18nを使う** — テキストはハードコーディングしない。全てi18nキーで管理。

6. **アクセシビリティ** — タブ切替は `aria-selected`、折りたたみは `aria-expanded` を使用。
